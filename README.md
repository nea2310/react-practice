# Real‑time Notifications App

Учебное приложение для демонстрации работы **WebSocket** в связке с **Redux Toolkit (RTK Query)**.  
Позволяет получать уведомления в реальном времени, отслеживать непрочитанные сообщения и видеть их количество в навигации.

---

## Технологии

**Клиент:**
- React 19 (функциональные компоненты, хуки)
- Redux Toolkit + RTK Query (управление состоянием и кэширование)
- React Router DOM (навигация)
- Vite (сборка и прокси)
- TypeScript

**Сервер:**
- Node.js + Express (HTTP‑эндпоинт)
- WebSocket (библиотека `ws`)
- CORS (для кросс-доменных запросов)

---

## Функциональность

- Получение начального списка уведомлений по HTTP при первом открытии страницы.
- Постоянное WebSocket‑соединение для приёма новых уведомлений в реальном времени.
- **Бейдж** с количеством **непрочитанных** уведомлений обновляется автоматически (даже когда вкладка «Уведомления» закрыта).
- При открытии вкладки «Уведомления» все непрочитанные сообщения выделяются **жёлтым** на **5 секунд**, затем выделение исчезает.
- Если вкладка закрыта – новые уведомления **накапливаются** в счётчике до следующего визита.
- Новые сообщения, пришедшие во время просмотра вкладки, также выделяются на 5 секунд.

## Установка и запуск

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/nea2310/react-practice.git
cd react-practice
```

### 2. Установите зависимости для клиента
bash

cd client
npm install

### 3. Установите зависимости для сервера
```bash

cd ../server
npm install
```

### 4. Запустите сервер (в отдельном терминале)
```bash

cd server
npm start
```

Сервер запустится на порту 5000 и будет доступен по адресу:

    HTTP: http://localhost:5000/api/initial-data

    WebSocket: ws://localhost:5000

### 5. Запустите клиент (в другом терминале)
```bash

cd client
npm run dev
```

Клиент запустится на порту 5173 (по умолчанию для Vite).
Откройте браузер и перейдите на http://localhost:5173.

    Примечание: В vite.config.ts настроен прокси для запросов на /api → http://localhost:5000. Благодаря этому клиент может обращаться к серверу без указания порта (например, fetch('/api/initial-data')).

## Как это работает (кратко)
Клиент (RTK Query)

    Эндпоинт getNotifications загружает начальные данные через HTTP (GET /api/initial-data).

    В onCacheEntryAdded создаётся WebSocket‑соединение к серверу.

    При получении сообщения update:

        Данные добавляются в кэш RTK Query.

        Диспатчится notificationsReceived – создаются метаданные (read: false, isNew: true).

        Если вкладка открыта – уведомление сразу помечается прочитанным (read: true), выделяется жёлтым на 5 секунд, затем выделение снимается.

        Если вкладка закрыта – уведомление остаётся непрочитанным и учитывается в бейдже.

Сервер (Express + WebSocket)

    HTTP‑эндпоинт /api/initial-data возвращает массив уведомлений в формате { id, text }.

    WebSocket‑сервер раз в 30 секунд генерирует новое уведомление и отправляет его всем подключённым клиентам.

    Каждое уведомление имеет уникальный id и текст.

Управление состоянием «прочитано/новое»

    Метаданные (read, isNew) хранятся в отдельном слайсе notificationsSlice (нормализованы с createEntityAdapter).

    Селектор selectUnreadCount подсчитывает количество записей с read: false.

    Бейдж в навигации обновляется автоматически через useSelector.


## API

#### HTTP

```GET /api/initial-data``` возвращает массив уведомлений:

```
[
    { "id": "init-1", "text": "Initial item 1" },
    { "id": "init-2", "text": "Initial item 2" }
]
```


#### WebSocket

    Соединение: 
```ws://localhost:5000```

    Сообщения от сервера (JSON):

```{
"type": "update",
"payload": [
        { "id": "item-3", "text": "Item 1234567890" }
    ]
}
```
